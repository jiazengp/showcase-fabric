# Item Sharing

<MinecraftVersion auto />

Item sharing allows players to showcase individual items with complete preservation of enchantments, NBT data, custom names, and all other properties.

## Basic Usage

### Simple Item Sharing

<CommandSyntax>/showcase item</CommandSyntax>

Share your currently held item with all players:

```
Player holds Diamond Sword with Sharpness V
Runs: /showcase item
Result: Creates shareable link in chat
```

### Targeted Sharing

<CommandSyntax>/showcase item [player]</CommandSyntax>

Share with specific players:

```bash
# Share with one player
/showcase item Steve

# Share with all players
/showcase item @a

# Share with nearest player
/showcase item @p
```

### Custom Duration

<CommandSyntax>/showcase item [player] [duration]</CommandSyntax>

Set custom expiration time:

```bash
# 5 minutes
/showcase item Steve 5m

# 2 hours
/showcase item @a 2h

# Permanent (until restart)
/showcase item Alex permanent
```

### With Description

<CommandSyntax>/showcase item [player] [duration] [description]</CommandSyntax>

Add custom description:

```bash
/showcase item @a 30m "My enchanted sword!"
/showcase item Steve 1h "Check out this rare item"
```

## Chat Keywords

### Quick Sharing

Type keywords in chat for instant sharing:

<Tabs items={["Basic Keywords", "Aliases", "Context"]}>
  <Tab value="Basic Keywords">
    **Primary Keywords**:
    ```
    [item] - Share held item
    [i]    - Short alias for [item]
    ```

    **Usage Examples**:
    ```
    "Check out my [item]!"
    → "Check out my [🗡️ Diamond Sword]!"

    "Look at this [i] I found!"
    → "Look at this [⛏️ Netherite Pickaxe] I found!"
    ```
  </Tab>

  <Tab value="Aliases">
    **Alternative Keywords** (configurable):
    ```yaml
    keywords:
      item:
        aliases: ["i", "hand", "held", "weapon", "tool"]
    ```

    All aliases work identically:
    ```
    [item] = [i] = [hand] = [held]
    ```
  </Tab>

  <Tab value="Context">
    **Smart Context Detection**:
    - Only works when holding an item
    - Respects permission requirements
    - Follows rate limiting rules
    - Maintains share duration settings

    **Failure Handling**:
    ```
    Empty hand + [item] → keyword ignored
    No permission + [item] → keyword ignored
    Rate limited + [item] → keyword ignored
    ```
  </Tab>
</Tabs>

## Configuration Options

### Share Settings

```yaml title="config/showcase/config.yml"
shares:
  item:
    # Enable/disable item sharing
    enabled: true

    # Default duration in minutes
    defaultDuration: 30

    # Maximum duration in minutes (0 = unlimited)
    maxDuration: 120

    # Cooldown between shares in seconds
    cooldown: 5

    # Custom duration permission required
    requirePermissionForDuration: false
```

### Chat Keyword Settings

```yaml
chatKeywords:
  keywords:
    item:
      enabled: true
      aliases: ["i", "hand", "held"]
      cooldown: 5

      # Custom formatting for chat display
      displayFormat: "[{item_name}]"
      colorCode: "§e"  # Yellow color
```

### Advanced Options

```yaml
itemSharing:
  # Show detailed tooltips
  showDetailedTooltips: true

  # Include technical NBT data
  includeTechnicalData: false

  # Maximum item name length in chat
  maxDisplayNameLength: 30

  # Truncate long names with "..."
  truncateLongNames: true

  # Show item icons in chat (requires resource pack)
  showItemIcons: false
```

## Best Practices

### For Players

<Callout type="tip">
**Sharing Tips**:
- Use descriptive titles for clarity
- Set appropriate durations (not too long)
- Consider your audience when sharing
- Use chat keywords for quick sharing
</Callout>

**Good Examples**:
```bash
✅ /showcase item @a 30m "Rare enchanted find!"
✅ Player: "Check out my new [item] from the raid!"
✅ /showcase item BuildTeam 1h "Tool for the project"
```

**Avoid**:
```bash
❌ /showcase item @a permanent "item"
❌ Spamming [item] in every message
❌ Sharing inappropriate or meaningless items
```

### For Server Admins

**Configuration Recommendations**:

**Small Servers** (< 20 players):
```yaml

    cooldown: 3

shares:
  item:

    cooldown: 1
```

## Troubleshooting

### Common Issues

<Callout type="error">
**"No item in hand"**: Must be holding an item to share
</Callout>

**Solution**: Hold the item you want to share before running the command.

<Callout type="error">
**Chat keywords not working**: `[item]` appears as plain text
</Callout>

**Solutions**:
1. Check permissions: `showcase.chat.item`
2. Verify you're holding an item
3. Check rate limits (may be temporarily blocked)
4. Ensure keywords are enabled in config

<Callout type="error">
**Share not appearing**: Command succeeds but no link in chat
</Callout>

**Debugging Steps**:
1. Check if share expired immediately
2. Verify cooldown hasn't blocked creation
3. Check share count limits
4. Review server logs for errors
